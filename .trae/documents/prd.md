## 1. 产品概述

中国象棋人机对弈应用，基于 Vue3 + Vite + LeaferJS 构建，提供完整的人机对局、复盘分析和沙盘推演功能。目标用户为象棋爱好者，核心价值在于提供符合标准象棋规则的智能对弈体验与棋局回顾能力。

## 2. 核心功能

### 2.1 功能模块

1. **主页**：开始对局入口、历史对局入口、难度选择
2. **对局页面**：棋盘绘制、棋子交互、AI对弈、菜单操作
3. **历史对局页面**：棋局列表、复盘分析、沙盘推演

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页 | 英雄区 | 应用标题、象棋主题装饰 |
| 主页 | 操作区 | "开始对局"按钮（弹出难度选择：简单/中等/困难）、"历史对局"按钮 |
| 对局页面 | 棋盘区 | LeaferJS 绘制9×10棋盘、楚河汉界、九宫格斜线 |
| 对局页面 | 棋子区 | 红黑双方16子，点击选中高亮，拖拽/点击走子，合法位置提示 |
| 对局页面 | 状态栏 | 显示当前轮次、被吃棋子、对局状态 |
| 对局页面 | 菜单栏 | 重新开始、退出本局、请求支招 |
| 对局页面 | 结局弹窗 | 胜/负/和提示，继续对局、复盘分析按钮 |
| 历史对局页面 | 对局列表 | 按时间倒序展示历史对局，显示日期、难度、结果 |
| 历史对局页面 | 复盘棋盘 | 棋盘展示当前步的棋局状态 |

| 历史对局页面 | 沙盘模式 | 进入沙盘推演，可自由走子，退出后恢复原棋局 |

## 3. 核心流程

用户打开应用 → 主页 → 点击"开始对局" → 选择难度 → 进入对局页面 → 人机交替走子 → 对局结束 → 选择"继续对局"或"复盘分析" → 复盘分析进入历史对局页面 → 逐步回放/沙盘推演 → 返回主页

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深木色(#8B4513) + 朱红(#C41E3A) + 墨黑(#1A1A2E)，体现传统中国象棋的古典韵味
- **辅助色**：金色(#D4A843)用于高亮选中、米白(#F5F0E8)用于背景
- **按钮风格**：圆角矩形，带有轻微阴影，红色主按钮 + 深色次要按钮
- **字体**：标题使用楷体/宋体风格，正文使用清晰易读的无衬线字体
- **布局风格**：居中对称布局，棋盘为核心焦点，底部操作栏
- **棋盘风格**：仿实木纹理底色，线条为深棕色，楚河汉界用书法字体

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页 | 英雄区 | 深色背景、象棋主题装饰图案、应用标题大字、副标题 |
| 主页 | 操作区 | 两个大按钮：朱红色"开始对局"、深色"历史对局" |
| 主页 | 难度弹窗 | 半透明遮罩、居中卡片、三个难度选项按钮 |

| 对局页面 | 棋子 | 圆形棋子、红色/黑色文字、选中时金色光圈、可走位置半透明提示 |
| 对局页面 | 状态栏 | 顶部显示轮次、被吃棋子排列 |
| 对局页面 | 菜单栏 | 底部三个按钮横排 |

| 历史对局页面 | 复盘控制 | 底部控制栏、播放按钮组 |

### 4.3 响应式设计

- 桌面优先设计，棋盘尺寸根据视口自适应
- 移动端适配：棋盘占满宽度，列表改为上下布局
- 触摸优化：棋子点击区域适当放大，支持触摸拖拽

### 4.4 走子规则

- **帅/将**：九宫格内一步直行
- **仕/士**：九宫格内一步斜行
- **相/象**：田字对角，不能过河，塞象眼不可走
- **马**：日字走法，蹩马腿不可走
- **车**：直线任意格，不可越子
- **炮**：直线任意格，吃子需隔一子（翻山）
- **兵/卒**：未过河只能前进，过河后可左右
- **将帅对面**：将帅不可在同一列无子对面
- **将军/将死**：被将军必须应将，无法应将则将死
